Complete Guide to Building a Blog with Tailwind CSS

By Rehan January 18, 2024 12 min read

Why Tailwind CSS for Blogs?

Tailwind CSS is perfect for blogs because:

  • Rapid prototyping - Build UI quickly without context switching
  • Responsive by default - Mobile-first approach built-in
  • Consistent design - Maintain visual coherence with utility classes
  • Small bundle size - PurgeCSS removes unused styles in production
  • Customizable - Easy to match your brand identity

Project Setup

1. Initialize Project

mkdir my-blog && cd my-blog
npm init -y

2. Install Tailwind CSS

npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

3. Configure Tailwind

Update tailwind.config.js:

module.exports = {
  content: ["./src/**/*.{html,js}"],
  theme: {
    extend: {
      colors: {
        brand: {
          light: '#f0f9ff',
          DEFAULT: '#0ea5e9',
          dark: '#0369a1',
        }
      },
      fontFamily: {
        'sans': ['Inter', 'system-ui', 'sans-serif'],
        'serif': ['Merriweather', 'Georgia', 'serif'],
      }
    },
  },
  plugins: [],
}

4. Add Base Styles

Create src/input.css:

@tailwind base;
@tailwind components;
@tailwind utilities;

@layer base {
  html {
    scroll-behavior: smooth;
  }
  
  body {
    @apply antialiased text-gray-800;
  }
  
  h1, h2, h3, h4 {
    @apply font-bold tracking-tight;
  }
}

Responsive Design

Tailwind makes responsive design effortless with breakpoint prefixes:

<div class="container mx-auto px-4">
  <!-- Grid changes based on screen size -->
  <div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-8">
    <!-- Blog posts -->
  </div>
  
  <!-- Responsive text sizes -->
  <h1 class="text-3xl sm:text-4xl md:text-5xl font-bold">
    Responsive Blog Title
  </h1>
  
  <!-- Responsive spacing -->
  <div class="mt-4 md:mt-8 lg:mt-12">
    <!-- Content with responsive margins -->
  </div>
  
  <!-- Show/hide elements based on screen size -->
  <div class="hidden lg:block">
    <!-- Desktop-only content -->
  </div>
  
  <div class="lg:hidden">
    <!-- Mobile-only content -->
  </div>
</div>

Dark Mode

Tailwind has built-in dark mode support:

<html class="dark">
<body class="bg-white dark:bg-gray-900 text-gray-900 dark:text-gray-100">

<!-- Dark mode toggle button -->
<button id="dark-toggle" class="p-2 rounded-lg bg-gray-100 dark:bg-gray-800">
  <svg class="w-5 h-5 text-gray-800 dark:text-yellow-400 hidden dark:block" fill="currentColor" viewBox="0 0 20 20">
    <!-- Sun icon -->
  </svg>
  <svg class="w-5 h-5 text-gray-800 dark:hidden" fill="currentColor" viewBox="0 0 20 20">
    <!-- Moon icon -->
  </svg>
</button>

<!-- Dark mode classes -->
<article class="bg-white dark:bg-gray-800 shadow dark:shadow-gray-900/30">
  <h3 class="text-gray-900 dark:text-white">Post Title</h3>
  <p class="text-gray-600 dark:text-gray-300">Post content...</p>
</article>

<script>
  const toggle = document.getElementById('dark-toggle');
  const html = document.documentElement;
  
  toggle.addEventListener('click', () => {
    html.classList.toggle('dark');
    localStorage.theme = html.classList.contains('dark') ? 'dark' : 'light';
  });
</script>

Performance Optimization

1. Purge Unused CSS

Ensure your tailwind.config.js is properly configured:

module.exports = {
  content: ['./**/*.html', './src/**/*.js'],
  // ... rest of config
}

2. Optimize Images

<!-- Use modern image formats -->
<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img 
    src="image.jpg" 
    alt="Description"
    loading="lazy"
    class="w-full h-auto"
  >
</picture>

  /* Inline critical CSS */

<link rel="stylesheet" href="styles.css" media="print" onload="this.media='all'">

Deployment

1. Build for Production

# Build Tailwind
npx tailwindcss -i ./src/input.css -o ./dist/output.css --minify

# Optimize with other tools if needed
npm run build

2. Deployment Options

Vercel (Recommended for blogs):

# Install Vercel CLI
npm i -g vercel

# Deploy
vercel --prod

Netlify:

# Add netlify.toml
[build]
  command = "npm run build"
  publish = "dist"

Static Hosting (GitHub Pages):

# Add to package.json
"scripts": {
  "deploy": "gh-pages -d dist"
}
