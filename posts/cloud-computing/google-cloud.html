Google Cloud Platform (GCP): infrastructure, serverless & data analytics

By Rehan February 12, 2026 13 min read

Why Google Cloud?

Cloud computing has become the backbone of modern technology, and Google Cloud Platform (GCP) stands out with its clean data architecture, global fiber network, and innovative AI/ML offerings. From startups to enterprises like Spotify, Twitter, and PayPal, GCP powers critical workloads. In this guide, we break down core GCP services, real-world use cases, and cost optimization – all under the cloud computing category.

1. Compute: VMs, containers & serverless

Compute Engine (IaaS)

Virtual machines on Google’s infrastructure. Custom machine types, sole-tenant nodes, and confidential VMs. Preemptible VMs save 60-80% for batch jobs.

# Create a VM with gcloud
gcloud compute instances create "rehan-instance" \
    --zone=us-central1-a \
    --machine-type=n2-standard-2 \
    --image-family=ubuntu-2204-lts \
    --image-project=ubuntu-os-cloud \
    --boot-disk-size=50GB

Google Kubernetes Engine (GKE)

Managed Kubernetes with auto-pilot, multi-cluster ingress, and GPU support. Industry-leading for container orchestration.

# Standard cluster with latest GKE
gcloud container clusters create rehan-cluster \
    --num-nodes=3 \
    --region=us-central1 \
    --enable-autoscaling \
    --min-nodes=1 --max-nodes=6 \
    --release-channel=stable

Cloud Run (serverless container)

Run stateless containers in a fully managed environment. Pay per request, no infra management. Perfect for APIs and microservices.

# Deploy a container to Cloud Run
gcloud run deploy rehan-service \
    --image gcr.io/my-project/my-app:latest \
    --platform managed \
    --region us-central1 \
    --allow-unauthenticated \
    --memory 2Gi

Cloud Functions (FaaS)

Event-driven, single-purpose functions. Integrated with Pub/Sub, Storage, Firestore.

// Node.js 20 example
exports.processImage = (event, context) => {
  const file = event.name;
  console.log(`New image uploaded: ${file}`);
  // image processing logic
};

2. Storage & Databases

Cloud Storage (GCS)

Object storage with strong consistency, multi-regional buckets, and lifecycle policies. Standard, Nearline, Coldline, Archive classes.

# Upload with gsutil
gsutil mb gs://rehan-blog-data/
gsutil cp report.csv gs://rehan-blog-data/

Cloud SQL & AlloyDB

Managed MySQL, PostgreSQL, SQL Server. AlloyDB for PostgreSQL – 4x faster than standard, ideal for demanding workloads.

Firestore & Bigtable

Firestore: serverless, scalable NoSQL document DB for mobile/web apps. Bigtable: petabyte-scale, low-latency NoSQL (HBase compatible).

3. Networking – built on the edge

VPC, Cloud CDN, Cloud Load Balancing

Global VPC, Cloud CDN (cache at Google’s 1400+ edges), multi-region load balancing. Private Google Access for internal communication.

# Create firewall rule
gcloud compute firewall-rules create allow-http \
    --network default \
    --allow tcp:80,tcp:443 \
    --source-ranges 0.0.0.0/0

Cloud Interconnect & Service Directory

Dedicated connectivity from on-prem to GCP. Service Directory – service discovery for microservices.

4. Data analytics & AI

BigQuery – serverless data warehouse

Analyze petabytes with SQL, built-in machine learning (BigQuery ML). No ops, fast, and separates storage/compute.

-- Example: top 10 customers
SELECT customer_id, SUM(amount) as total
FROM `my-project.sales.transactions`
WHERE date >= '2026-01-01'
GROUP BY customer_id
ORDER BY total DESC
LIMIT 10;

Dataflow & Dataproc

Stream/batch processing with Apache Beam (Dataflow) and managed Spark/Hadoop (Dataproc).

Vertex AI

Unified ML platform: AutoML, custom training, feature store, and model monitoring. Deploy models with one click.

# Deploy model to endpoint
gcloud ai endpoints deploy-model 123456 \
    --region=us-central1 \
    --model=projects/rehan-blog/models/recommender \
    --display-name=prod-v2

5. DevOps & developer tools

Cloud Build & Artifact Registry

CI/CD fully managed. Artifact Registry: next-gen container/dependency storage.

# Build and push with Cloud Build
gcloud builds submit --tag us-central1-docker.pkg.dev/my-project/repo/image:tag

Cloud Deploy, Source Repositories, and Logging

Git hosting, SRE tools. Cloud Monitoring and Error Reporting. Operations Suite (formerly Stackdriver).

6. Practical: cost governance & best practices

  • Labels & Budget alerts: assign labels (e.g., environment, cost-center). Set budget thresholds and alerts.
  • Committed use discounts & Spot VMs: save 30-70% for steady-state or fault-tolerant workloads.
  • IAM & organization policies: principle of least privilege. Use custom roles, deny policies.
  • Security: VPC Service Controls, Shielded VMs, CMEK for encryption.
# Label a project
gcloud compute instances create demo --labels=team=rehan,env=staging,app=blog

7. Real-world case: migration to GKE and Cloud Run

A SaaS company migrated from monolithic VMs to GKE (stateful) + Cloud Run (stateless). Results:
→ 45% infra cost reduction
→ 5x faster deploys
→ autoscaling from 0 during idle hours (Cloud Run).

8. Next steps: certification & hands-on

Google Cloud certifications (Associate Cloud Engineer, Professional Cloud Architect) validate skills. Use free tier – always free products: Cloud Build, BigQuery 1TB/month, Cloud Run 2M requests. Follow labs on Google Cloud Skills Boost.

Conclusion: why GCP fits modern cloud strategy

Google Cloud provides a consistent, open, and sustainable platform. Its strengths lie in data, machine learning, and serverless. For developers, startups, and enterprises, GCP simplifies operations while scaling with demand.
