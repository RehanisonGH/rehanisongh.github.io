Microsoft Azure Guide: Core Services, Architecture & Best Practices (2024)

By Rehan February 18, 2024 14 min read

Introduction

Microsoft Azure has grown into one of the most comprehensive cloud platforms, offering over 200 products and cloud services. Whether you're migrating legacy workloads, building cloud-native applications, or deploying AI solutions, Azure provides the infrastructure, platform, and tools. This guide covers everything a cloud practitioner needs — from core compute and storage to governance, cost optimization, and real-world architectures. No dark themes, just plain light-mode knowledge.

Why Azure? The Enterprise-Ready Cloud

With a global footprint of 60+ regions, Azure leads in hybrid capabilities, compliance certifications, and seamless integration with Microsoft products. Below we break down the essential categories every developer and architect should master.

1. Azure Compute: VMs, App Service, Containers & Serverless

Azure offers multiple compute models. Virtual Machines (IaaS) give you full control; App Service provides a fully managed PaaS for web apps; Azure Kubernetes Service (AKS) simplifies container orchestration; and Azure Functions delivers event-driven serverless compute.

Quick comparison:

  • Virtual Machines – lift-and-shift, custom OS, scaling sets.
  • App Service – auto-scaling, CI/CD integration, Windows/Linux.
  • AKS – managed Kubernetes, DevOps friendly, Azure AD integration.
  • Functions – consumption plan, triggers (blob, queue, HTTP).
// Example: Deploy a Node.js web app to Azure App Service via Azure CLI
az group create --name myResourceGroup --location eastus
az appservice plan create --name myPlan --resource-group myResourceGroup --sku B1 --is-linux
az webapp create --resource-group myResourceGroup --plan myPlan --name myUniqueAppName --runtime "NODE:18-lts"
az webapp deployment source config --name myUniqueAppName --resource-group myResourceGroup --repo-url https://github.com/user/repo --branch main

2. Azure Storage: Blob, Disk, Files & Data Lake

Azure Storage is massively scalable, secure, and offers multiple data services. Blob storage for unstructured data, Managed Disks for VM disks, Azure Files for SMB shares, and Data Lake Gen2 for big data analytics.

Hot, Cool, and Archive tiers

Optimize cost by moving infrequently accessed data to Cool or Archive. Lifecycle management policies automate transitions.

# Upload a file to Azure Blob Storage using AzCopy
azcopy copy 'C:\local\file.txt' 'https://mystorageaccount.blob.core.windows.net/mycontainer/file.txt?[SAS_TOKEN]'

3. Networking: Virtual Network, Load Balancer, CDN, and Firewall

Secure connectivity is core to Azure. Build isolated networks with Azure Virtual Network (VNet), distribute traffic with Azure Load Balancer or Application Gateway, and protect applications with Azure Firewall and DDoS Protection. Global reach via Azure Front Door and CDN.

Typical hub-spoke topology

  • Hub VNet: shared services (firewall, management).
  • Spoke VNets: isolated workloads (prod, dev).
  • VNet peering with traffic flow through hub.

4. Azure Databases: SQL, Cosmos DB, PostgreSQL, MySQL

Azure SQL Database – fully managed SQL Server; Cosmos DB – globally distributed multi-model database; Azure Database for PostgreSQL/MySQL – community editions with enterprise manageability. Also Azure Cache for Redis for low-latency caching.

// Create Cosmos DB (NoSQL API) via Azure CLI
az cosmosdb create --name mycosmosdb --resource-group myResourceGroup --kind GlobalDocumentDB
az cosmosdb sql database create --account-name mycosmosdb --name ProductsDB --resource-group myResourceGroup

5. Identity & Security: Azure AD, RBAC, Key Vault

Azure Active Directory is the backbone of identity, providing SSO, MFA, and conditional access. Role-Based Access Control (RBAC) grants granular permissions. Key Vault safeguards secrets, keys, and certificates. For workloads, Managed Identities eliminate credential storage.

Best practice: always use managed identities

// Example: enable system-assigned managed identity for VM
az vm identity assign --resource-group myResourceGroup --name myVM

Architecture deep-dive: Microservices on Azure

Let's combine services to build a resilient, scalable solution. Consider an e‑commerce backend using AKS, Cosmos DB, Redis Cache, API Management, and Event Grid.

Components:

  • AKS – containerized order service, product catalog.
  • Cosmos DB – globally replicated product inventory.
  • Azure Cache for Redis – session store, rate limiting.
  • API Management – expose APIs with policies and subscription keys.
  • Event Grid – serverless event routing (order created → email function).
// AKS deployment snippet (deployment.yaml)
apiVersion: apps/v1
kind: Deployment
metadata:
  name: order-api
spec:
  replicas: 3
  selector:
    matchLabels:
      app: order-api
  template:
    metadata:
      labels:
        app: order-api
    spec:
      containers:
      - name: order-api
        image: myacr.azurecr.io/order-api:v1
        ports:
        - containerPort: 8080
        env:
        - name: COSMOS_CONNECTION
          valueFrom:
            secretKeyRef: 
              name: cosmos-secret
              key: connection

6. Azure DevOps & Infrastructure as Code

Adopt DevOps with Azure Repos, Pipelines, and Bicep/ARM templates. Bicep is a domain-specific language for declarative Azure deployments, simpler than raw ARM JSON.

// Bicep sample – deploy storage account and blob container
param storageName string = 'stg${uniqueString(resourceGroup().id)}'
param containerName string = 'logs'

resource sa 'Microsoft.Storage/storageAccounts@2023-01-01' = {
  name: storageName
  location: resourceGroup().location
  sku: { name: 'Standard_LRS' }
  kind: 'StorageV2'
}

resource container 'Microsoft.Storage/storageAccounts/blobServices/containers@2023-01-01' = {
  name: '${sa.name}/default/${containerName}'
}

Cost Management and Governance

Azure provides robust tools to control spending: Azure Policy to enforce rules, Management Groups + Subscriptions for organizational hierarchy, and Cost Management + Billing for budgets and anomaly alerts. Use tags to attribute costs.

Recommendations for cost efficiency:

  • Reserved Instances for steady-state VMs (up to 72% savings).
  • Azure Hybrid Benefit – reuse Windows Server/SQL Server licenses.
  • Scale down/up based on schedule (DevTest labs).
  • Use Spot VMs for batch workloads.

Migration: On‑prem to Azure – the 5 R's

Leverage Azure Migrate to assess and migrate. Common strategies: Rehost (IaaS), Refactor (PaaS), Rearchitect (cloud-native), Rebuild, or Replace (SaaS). Example: moving SQL Server to Azure SQL Managed Instance reduces maintenance.

Case snapshot: logistics company

After moving legacy warehouse management to Azure VMs and refactoring the reporting layer to Azure Synapse, they achieved 40% lower TCO and near-zero downtime during peak.

Azure for AI/ML and IoT

Azure Machine Learning – end-to-end platform; Cognitive Services – prebuilt vision/speech/language APIs; Azure IoT Hub – bi-directional communication with millions of devices. These services fit seamlessly into the ecosystem.

// Use Azure Cognitive Services Text Analytics client library (Python)
from azure.ai.textanalytics import TextAnalyticsClient
from azure.core.credentials import AzureKeyCredential

endpoint = "https://mytextanalytics.cognitiveservices.azure.com/"
key = "my_key"
credential = AzureKeyCredential(key)
text_analytics_client = TextAnalyticsClient(endpoint, credential)

documents = ["Azure is a comprehensive and scalable cloud platform."]
response = text_analytics_client.extract_key_phrases(documents)
for doc in response:
    print(doc.key_phrases)

Certification roadmap for Azure professionals

Microsoft Learn offers role-based certifications. Start with AZ-900 (Azure Fundamentals), then AZ-104 (Administrator) or AZ-204 (Developer). For architects: AZ-305. Keep labs active.

Best practices: reliability & performance

  • Availability Zones for high‑availability.
  • Auto‑scale rules based on metrics (CPU, queue length).
  • Azure Monitor + Application Insights – detect anomalies and set alerts.
  • Backup vaults and Site Recovery for DR.

Conclusion: why Azure now?

Microsoft Azure continues to innovate, with recent advancements in Azure OpenAI Service, confidential computing, and sustainable data centers. Whether you're a startup or a Fortune 500, Azure's hybrid consistency, developer velocity, and responsible AI principles make it a powerful choice. This guide covered the pillars – but the cloud evolves fast. Keep experimenting, use the free account, and join the community.
