Mobile UI Design: From Gestures to Components — Complete Guide 2024

By Rehan February 18, 2024 9 min read

Introduction

Mobile UI design isn’t just about smaller screens — it’s about thumbs, context, and platform conventions. With over 7 billion mobile devices worldwide, crafting interfaces that feel native and effortless is no longer optional. Whether you're building with SwiftUI, Jetpack Compose, or React Native, the core principles remain. In this guide, we break down modern mobile UI design patterns, adaptive layouts, and the subtle differences between iOS and Android that every mobile developer must know.

1. Platform personality: iOS vs Android

A button on iOS feels different from Android. Successful mobile UI respects these human interface guidelines without reinventing the wheel.

iOS (Human Interface Guidelines)

  • Depth & motion: Translucent navigation bars, blurred backgrounds, and smooth transitions.
  • Top bar emphasis: Navigation bars at the top, back buttons as "< Title".
  • Bottom sheets: Cards that slide from bottom — common in Maps, Apple Music.

Android (Material Design 3)

  • Bold color & elevation: Surface tints, shadows as communication.
  • Bottom navigation: Usually 5 icons, labels always visible.
  • Back navigation: System back + contextual up button.
// Example: Platform specific icon sets in React Native
import { Platform } from 'react-native';

const ShareIcon = () => (
  <Icon 
    name={Platform.OS === 'ios' ? 'ios-share' : 'md-share'} 
    size={24} 
  />
);

2. Thumb zone & touch targets

People hold phones with one hand, using their thumb. The "easy reach" zone is the bottom half of the screen.

Key rules:

  • Minimum touch target: 44x44pt (iOS) / 48x48dp (Android).
  • Place primary actions at the bottom: FABs, CTAs, tab bars.
  • Avoid hover interactions: They don't exist on touch.

Thumb comfort map

Center and bottom-right are natural; top-left requires hand readjustment. Critical actions like "delete" or "submit" should never be in the hard-to-reach top corners.

3. Navigation patterns that scale

Tab bar / Bottom navigation

Most common pattern for primary navigation. Should contain 3–5 destinations of equal importance.

Gesture-based navigation

Swipe back (iOS), swipe to delete, pull to refresh. Make gestures discoverable — use subtle hints.

// SwiftUI — custom swipe action
List {
    Text("Message")
        .swipeActions {
            Button("Delete", role: .destructive) {
                // delete action
            }
            Button("Archive") {
                // archive
            }
            .tint(.blue)
        }
}

Modal vs Push

Use modals for self-contained tasks (create, compose, edit). Use push for drilling down into details.

4. Mobile typography & readability

Small screens demand larger default font sizes and high contrast.

  • Body text: 16–17pt (iOS) / 16sp (Android). Never below 14pt for long text.
  • Line height: 1.4–1.5x for readability.
  • Dynamic type / font scaling: Respect user's accessibility settings.

5. Component deep dive: Cards, Lists & Forms

Cards

Should be modular, with clear separation. Avoid nested touch targets.

Forms on mobile

Pain point for users. Optimize by:

  • Showing numeric keyboard for PINs/phone numbers.
  • Inline validation after input, not on submit.
  • Grouping related fields.
// Jetpack Compose — optimized text field
TextField(
    value = phone,
    onValueChange = { phone = it },
    keyboardOptions = KeyboardOptions(
        keyboardType = KeyboardType.Phone,
        imeAction = ImeAction.Next
    ),
    singleLine = true
)

6. Adaptive & responsive layouts

Mobile isn't just one screen size — foldables, tablets, and landscape mode require adaptive UIs.

Strategies:

  1. ConstraintLayout (Android) / Auto Layout (iOS): Dynamic spacing.
  2. SlidingPaneLayout / UISplitViewController: Master-detail for tablets.
  3. Window size classes (Material 3): Compact, medium, expanded.

7. Dark mode & theming

Not just black background. Implement semantic colors, not hardcoded hex.

// Flutter — theme awareness
Container(
  color: Theme.of(context).colorScheme.surface,
  child: Text(
    'Adaptive text',
    style: Theme.of(context).textTheme.bodyLarge,
  ),
);

8. From design to code: Figma workflow

Bridging the gap with tools like Zeplin, Figma Inspect, or Plugins.

  1. Design tokens: export colors, spacing, typography as JSON.
  2. Components: match names with code component library.
  3. Auto layout: translates to Flexbox / ConstraintLayout.

9. Performance & UI rendering

Smooth UI = 60fps (or 120Hz on ProMotion). Avoid:

  • Complex view hierarchies.
  • Unnecessary re-composition/re-layout.
  • Overdraw (hidden backgrounds).

Profile tools

  • Layout Inspector (Android Studio)
  • Xcode View Debugger
  • React Native Perf Monitor

10. Accessibility first

Over 1 billion people have disabilities. Accessible UI is better UI for everyone.

  • VoiceOver / TalkBack: every element must have label and trait.
  • Sufficient contrast: 4.5:1 for normal text.
  • Not only color: convey meaning with icons + text.
// iOS — accessibility label
Button(action: send) {
    Image(systemName: "paperplane")
}
.accessibilityLabel("Send message")
.accessibilityHint("Double tap to send")

Case study: Redesigning a food delivery app

Problem: Users abandoned cart because "place order" button was at top-right.

Solution: Moved CTA to bottom bar with total price. Checkout completion ↑22%.

Lesson: Thumb zone optimization directly impacts conversion.

Future: Gestures, Haptics & Motion

iOS 17 and Android 14 push for richer haptic feedback and micro-interactions. Use haptics sparingly to confirm actions, not every tap.

Conclusion

Mobile UI is a discipline where psychology meets engineering. By respecting platform idioms, optimizing for thumbs, and writing adaptive code, you create apps that feel intuitive and polished. Start by auditing one screen in your current project: is the primary action within easy reach? Does it respect the system font size? Small changes lead to massive UX wins.

Next steps: Download Apple’s HIG and Material Design 3 guidelines — they’re free and constantly updated. Pair them with platform-specific code samples, and you’re on your way to becoming a mobile UI expert.
