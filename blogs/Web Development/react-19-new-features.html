1. Actions API

The new Actions API represents a fundamental shift in how we handle data mutations in React. It provides a first-class way to manage server actions, form submissions, and asynchronous state updates.

Key Benefits

  • Simplified form handling with automatic state management
  • Built-in pending, error, and success states
  • Seamless integration with React's concurrent features
  • Reduced boilerplate code for async operations
// React 19 - Using the new Actions API
import { useActionState, useFormStatus } from 'react';

function AddToCartButton({ productId }) {
  // useActionState handles async actions with built-in state
  const [state, formAction, isPending] = useActionState(
    async (prevState, formData) => {
      const response = await addToCart(productId, formData);
      if (response.success) {
        return { message: 'Added to cart!' };
      } else {
        return { message: 'Failed to add item', error: response.error };
      }
    },
    { message: '' }
  );

  return (
    <form action={formAction}>
      <button type="submit" disabled={isPending}>
        {isPending ? 'Adding...' : 'Add to Cart'}
      </button>
      {state.message && <p>{state.message}</p>}
    </form>
  );
}

2. Optimistic Updates

React 19 introduces built-in support for optimistic updates, making it easier to provide instant UI feedback while asynchronous operations are processing in the background.

// Using the new useOptimistic hook
import { useOptimistic } from 'react';

function MessageList({ messages, sendMessage }) {
  // useOptimistic provides immediate UI updates
  const [optimisticMessages, addOptimisticMessage] = useOptimistic(
    messages,
    (state, newMessage) => [
      ...state,
      { ...newMessage, sending: true }
    ]
  );

  async function handleSubmit(formData) {
    const newMessage = { text: formData.get('text') };
    
    // Immediately update UI optimistically
    addOptimisticMessage(newMessage);
    
    // Then send to server
    await sendMessage(newMessage);
  }

  return (
    <>
      {optimisticMessages.map((msg, i) => (
        <div key={i} className={msg.sending ? 'sending' : ''}>
          {msg.text} {msg.sending && <small>(Sending...)</small>}
        </div>
      ))}
      <form onSubmit={handleSubmit}>
        <input name="text" />
        <button type="submit">Send</button>
      </form>
    </>
  );
}

3. Enhanced Hooks

React 19 brings improvements to several existing hooks and introduces new ones for better developer experience and performance.

Hook React 18 React 19 Enhancement
use Not available New hook that can read resources (promises, context) conditionally
useMemo Manual dependency arrays Compiler can auto-optimize in many cases
useEffect Runs after render Better scheduling with concurrent features
useCallback Manual dependency arrays Reduced need with Actions API patterns 
// New `use` hook for reading promises and context
import { use } from 'react';

function UserProfile({ userPromise }) {
  // `use` can be called conditionally, unlike other hooks
  try {
    const user = use(userPromise);
    return <h1>{user.name}</h1>;
  } catch (error) {
    return <div>Error loading user</div>;
  }
}

4. Document Metadata Tags

React 19 allows you to render metadata tags (title, meta, link) directly from any component, which is especially useful for SEO and social sharing.

// Setting metadata from any component
function BlogPost({ post }) {
  return (
    <>
      <!-- These work anywhere in the component tree -->
      <title>{post.title}</title>
      <meta name="description" content={post.excerpt} />
      <meta property="og:image" content={post.thumbnail} />
      <link rel="canonical" href={post.canonicalUrl} />
      
      <article>
        <h1>{post.title}</h1>
        <div>{post.content}</div>
      </article>
    </>
  );
}

This feature eliminates the need for external libraries like React Helmet in many cases, as React 19 now handles metadata tags natively and ensures they're properly placed in the document head.

5. Performance Improvements

React 19 includes several under-the-hood improvements that enhance performance without requiring code changes.

Key Performance Updates

  • Compiler optimizations: Automatic memoization in more cases
  • Reduced hydration mismatches: Better error messages and recovery
  • Faster initial render: Improvements to the concurrent renderer
  • Smaller bundle size: Tree-shaking improvements for server components
