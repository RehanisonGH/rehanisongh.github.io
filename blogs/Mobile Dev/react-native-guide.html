React Native GitHub Guide: Complete Setup & Workflow

Learn how to efficiently manage React Native projects with GitHub, from initial setup to CI/CD and collaboration best practices.

Introduction

React Native has revolutionized mobile app development by allowing developers to build native mobile apps using JavaScript and React. When combined with GitHub, you have a powerful workflow for version control, collaboration, and continuous integration. This guide will walk you through setting up a React Native project with GitHub and implementing best practices for efficient development.

Initial Project Setup

Let's start by creating a new React Native project and connecting it to GitHub.

Step 1: Create a New React Native Project

Use the React Native CLI to initialize a new project:

npx react-native init MyReactNativeApp

Step 2: Initialize Git Repository

Navigate to your project directory and initialize Git:

cd MyReactNativeApp
git init
git add .
git commit -m "Initial commit"

Step 3: Create GitHub Repository and Push

Create a new repository on GitHub and push your code:

git remote add origin https://github.com/yourusername/MyReactNativeApp.git
git branch -M main
git push -u origin main

Pro Tip

Always create a .gitignore file tailored for React Native projects. The React Native CLI creates one for you, but ensure it includes:

  • Node modules (node_modules/)
  • Build artifacts (android/app/build, ios/build)
  • Environment files (.env, but not .env.example)
  • OS-specific files (DS_Store, thumbs.db)

GitHub Repository Structure

A well-organized repository structure makes collaboration easier. Here's a recommended structure for React Native projects:

MyReactNativeApp/
├── android/ # Android native code
├── ios/ # iOS native code
├── src/ # Main application source
│ ├── components/ # Reusable components
│ ├── screens/ # Screen components
│ ├── navigation/ # Navigation setup
│ ├── utils/ # Utility functions
│ └── assets/ # Images, fonts, etc.
├── __tests__/ # Test files
├── .github/ # GitHub workflows and templates
├── .gitignore # Git ignore rules
├── package.json # Dependencies and scripts
└── README.md # Project documentation

Git Branch Strategy

Adopting a consistent branching strategy is crucial for team collaboration. We recommend the GitHub Flow approach:

Main Branches

  • main - Production-ready code
  • develop - Integration branch for features (optional)

Supporting Branches

  • feature/* - New features (e.g., feature/user-auth)
  • bugfix/* - Bug fixes (e.g., bugfix/login-crash)
  • hotfix/* - Critical production fixes (e.g., hotfix/security-patch)
  • release/* - Release preparation (e.g., release/v1.2.0)
# Create a new feature branch
git checkout -b feature/user-profile main

# After development, push to GitHub
git push -u origin feature/user-profile

# Create a Pull Request on GitHub
# After PR approval and merge, delete the branch
git branch -d feature/user-profile

GitHub Actions for React Native

Automate your React Native workflow with GitHub Actions. Here's a basic CI configuration:

# .github/workflows/react-native-ci.yml
name: React Native CI

on:
push:
branches: [ main, develop ]
pull_request:
branches: [ main ]

jobs:
test:
runs-on: ubuntu-latest
steps:
- uses: actions/checkout@v3

- name: Setup Node.js
uses: actions/setup-node@v3
with:
node-version: '18'

- name: Install dependencies
run: npm ci

- name: Run tests
run: npm test -- --passWithNoTests

- name: Lint code
run: npm run lint

Important Note

For React Native, you'll need additional setup for building Android/iOS apps in CI. Consider using React Native Setup Action or creating custom workflows for each platform.

Essential GitHub Features for React Native

1. Pull Request Templates

Create .github/PULL_REQUEST_TEMPLATE.md to standardize PR descriptions:

## Description
Brief description of the changes

## Type of change
- [ ] Bug fix
- [ ] New feature
- [ ] Breaking change
- [ ] Documentation update

## Checklist
- [ ] My code follows the style guidelines
- [ ] I have performed a self-review of my code
- [ ] I have added tests that prove my fix is effective
- [ ] New and existing unit tests pass locally

2. Issue Templates

Use issue templates for bugs, features, and documentation requests in .github/ISSUE_TEMPLATE/.

3. GitHub Projects

Use GitHub Projects to track your React Native app development with automated kanban boards linked to issues and PRs.

Managing Secrets and Environment Variables

Never commit sensitive data to your repository. Use GitHub Secrets and environment files:

# .env.example (commit this)
API_URL=https://api.example.com
GOOGLE_MAPS_API_KEY=your_key_here

# .env (add to .gitignore)
API_URL=https://api.production.com
GOOGLE_MAPS_API_KEY=production_key_here

In GitHub Actions, use repository secrets to store sensitive data and access them in workflows:

# In GitHub Actions workflow
- name: Build Android
env:
KEYSTORE_PASSWORD: ${{ secrets.KEYSTORE_PASSWORD }}
KEYSTORE_ALIAS: ${{ secrets.KEYSTORE_ALIAS }}

Release Management

Use GitHub Releases to manage your React Native app versions:

# Create a tag for a new release
git tag -a v1.0.0 -m "Release version 1.0.0"
git push origin v1.0.0

# Then create a release on GitHub with:
# - Release notes
# - APK/IPA attachments
# - Source code zip

Conclusion

Integrating GitHub into your React Native workflow provides powerful tools for collaboration, automation, and project management. By following the practices outlined in this guide, you'll establish a robust development pipeline that scales with your project's complexity.

Remember that every team is different, so adapt these practices to fit your specific needs. The key is consistency—once you establish a workflow, ensure all team members follow it for smooth collaboration.
